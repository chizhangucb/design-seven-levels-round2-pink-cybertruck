Screenshots & references

Show, don't tell. One reference image transfers visual intent almost losslessly — paragraphs of adjectives can't.

The technique

Grab a design you love from godly.website, Land-book, Awwwards, or Dribbble, and attach the screenshot. The model reads structure, spacing, palette, and mood straight off the pixels.

Round 1 needed a follow-up here to make the nav clickable. This round it sat loaded and unused — both models shipped functional nav anchors on the first pass, and Opus even self-checked its page in a browser.

The prompts that did it

Restyle toward the reference
hey there, I would like you to update the website using this image as a reference style (only this one) for how i would likeThe website you look @reference-for-pink-cybertruck-web.png keep pink-cybertruck-web-level-1.html untouched. name the file as pink-cybertruck-web-level-2.html

Both models restyled the level-1 page toward the reference screenshot. Level 1 was checksum-verified untouched in both runs — the keep-untouched rule now rides up front, and round 1's overwrite (Opus clobbered level 1 here last time) did not recur. The scripted 'make the nav clickable' follow-up was never needed: both models' anchors worked.

Cost & time

What changed vs level 1

  • Layout stops being generic — it inherits the reference's structure and rhythm.
  • The palette and type treatment shift decisively toward the reference.
  • Round 1's lesson, applied: the save-as rule now leads the prompt — and the overwrite didn't recur.

What this level taught

  • An image speaks a thousand words — literally true for design prompting.
  • State file rules up front; prevention beats the recovery follow-up.
  • Staged follow-ups can go unused — both models made the nav functional without being asked twice.
