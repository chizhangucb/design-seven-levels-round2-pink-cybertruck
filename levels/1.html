Level 1 of 7

Grab and go

Treat Claude like a chatbot: no references, no skills, no assets. The baseline everything else improves on.

The technique

This is how most people use Claude for websites — a single plain-language ask. You get a competent, generic 'AI website': serviceable layout, safe typography, no point of view.

It's worth running deliberately, because every later level is measured against it. One trick that pays off immediately: name the output file in the prompt, so each level stays immutable evidence.

The prompts that did it

The entire level, one prompt
Claude, I want you to build for me a simple website or just a nice looking website that is cool about Cybertrucks that are pink. Build me one of those website. name the file as pink-cybertruck-web-level-1.html

Fable produced a light rose one-pager with a frosted-glass nav — and a fan-concept disclaimer, unprompted. Opus 4.8 went dark: neon magenta on steel, with a color-picker gimmick that repaints the truck. Both one-shot it in under two minutes.

Cost & time

What this level taught

  • A bare prompt is a floor, not a ceiling — and knowing the floor is useful.
  • Model personality shows up even at level 1: light editorial (Fable) vs dark neon (Opus) from the identical prompt.
  • Two API calls each — the baseline is fast and cheap; everything above it is deliberate spend.
