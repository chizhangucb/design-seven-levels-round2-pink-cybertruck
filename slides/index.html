Leveling Up Website Quality in Claude Code — Round 2: the Pink Cybertruck

Seven levels · two models · the experiment ran itself.

Why this talk

Most Claude-built websites look generic — and it isn't the model's fault.

  • No references, no skills, no data — "Mozart with no piano."
  • Source of the ladder: Jack Roberts' 7 Levels of Building Claude Code Websites.
  • Round 1 replicated it with receipts. Round 2 asks: does it replicate again?

The experiment

One pink Cybertruck. The same prompts, run twice.

  • Fable 5 and Opus 4.8, identical prompts, parallel folders, same references and skills.
  • Every level's HTML frozen the moment it was built — SHA-256 checksummed, per-level metrics mined from transcripts.
  • New this round: the experiment ran itself. One orchestrator session drove all 14 level-sessions headlessly (claude -p).

Ground rules

Treat prompting experiments like experiments.

  • Named output files per level; "keep level-(N−1) untouched" up front in every prompt.
  • One variable changed per level; scripted follow-ups only — no improvising.
  • Disclosed deviations (a)–(f) in the runbook: new L4 component, symmetric L3/L4 inputs, new L6/L7 targets, orchestrated mode.

Level 4 · Result

Opus debugs the component it was handed

  • Opus caught the source's 0–255 vertex-color bug: "three.js reads color buffers as 0–1, so those dots were silently clamped to pure white — I used proper 0–1 values." The round-2 echo of round 1's memory-leak find.
  • Both went CDN — but Opus deliberately picked the classic three.js build so the file works when double-clicked from disk (file:// blocks module scripts); Fable used a CDN import and added a prefers-reduced-motion static frame.
  • Both kept the source's full-viewport fixed inset-0 wrapper — exactly as predicted — and both needed the one scripted rescope follow-up.

Replication verdict

Same ladder, different product: the ladder held.

  • Replicated: monotonic quality climb · L3 file-size jump · L7 design-language convergence (in voice, not in brand name) · L4's "models keep the fixed-viewport wrapper" prediction hit both models.
  • Didn't replicate: round 1's verification asymmetry (both models verify now).
  • Didn't recur: the overwrite incident — prevented by design this round.

The orchestrated run

An agent ran the experiment on two other agents

  • One orchestrator drove 14 level-sessions + 4 scripted follow-up resumes, logged it, checksummed it — and built this deck.
  • ~1h45m of pipeline wall-clock (23:32–02:08 UTC, minus one ~50-min human pause) · zero recovery prompts · zero clobbered files.
  • Interventions: one human resume (a permissions pause on the orchestrator side, not the experiment) and one preflight environment fix (Firecrawl CLI wasn't authenticated — found the key in ~/.zshrc). Every follow-up sent was pre-scripted, fired on its stated condition.

The meta-ladder

Imitate → Codify → Generalize

  • Imitate (round 1) — followed the video by hand: interactive, improvised. Fable 5: $102 / ~96 min, one overwrite incident.
  • Codify — Claude summarized its own run into an interactive tutorial, a fixed prompt pack, and a metrics methodology.
  • Generalize (round 2) — the framework ran itself on a new product: $37 / ~43 min, zero improvised prompts, zero incidents — and it built this deck.
  • That's Level 7 applied to the process itself: extract the process-language, re-render it on a new product.

Verdict

Held in round 2: both models climbed from the same generic PINKTRUCK baseline to distinct, credible design-language pages. The technique ladder did the lifting; the model chose the style of the climb.

Meta-lessons

Five inequalities to remember

  • References > adjectives
  • Skills > instructions
  • Research > invention
  • Verification > hope
  • New this round: Orchestration > babysitting

Tricks & gotchas

Field notes, round 2

  • Named outputs + "keep level-(N−1) untouched" up front — it worked: no overwrite this round.
  • One live session per directory; log session IDs as you go.
  • The stale-frame screenshot artifact bit again — this time the orchestrator's own browser pane while verifying L4; Playwright settled it, same as round 1.
  • Models wander: Fable wrote its L7 kit to ~/Desktop instead of the working dir.
  • Preflight pays: an unauthenticated Firecrawl CLI was caught before it could stall L6.
